Tutorial: Git Fork y Pull Requests

Cómo hacer un fork a un repositorio y colaborar en él

El fork es una forma de crear una copia personal de un repositorio ajeno. Esto te permite experimentar con cambios sin afectar el proyecto original.

Pasos para hacer un fork:

  1. Entra al repositorio que quieres hacer fork.
  2. Haz clic en el botón "Fork" en la esquina superior derecha.
  3. GitHub creará una copia del repositorio en tu cuenta.
  4. Ahora tienes tu propia copia del repositorio donde puedes hacer cambios.

Cómo crear ramas en Git

Las ramas en Git permiten trabajar en diferentes características o correcciones de manera aislada.

Configuración Inicial

Asegúrate de tener Git instalado. Puedes verificar tu versión con:

git --version

Pasos para crear y usar ramas:

  1. Verifica las ramas existentes: 
    git branch
  2. Crea una nueva rama: 
    git branch nombre-de-la-rama
  3. Cambia a la nueva rama: 
    git checkout nombre-de-la-rama
  4. Realiza cambios y confírmalos: 
    git add .
                    git commit -m "Descripción de los cambios realizados"

Cómo utilizar Pull Requests en GitHub

Un Pull Request (PR) es una forma de solicitar la revisión y fusión de tus cambios en el proyecto principal.

Pasos para crear un Pull Request:

  1. Ve a tu repositorio forkeado en GitHub.
  2. Haz clic en "New Pull Request".
  3. Selecciona tu rama y la rama destino del repositorio original.
  4. Escribe una descripción clara de los cambios realizados.
  5. Haz clic en "Create Pull Request".

Revisión y discusión del Pull Request:

Otros miembros del equipo podrán revisar tu código y dejar comentarios.

Aceptación del Pull Request:

Cuando el PR sea aprobado, se puede fusionar con la rama principal.
