Tutorial: Cómo crear ramas en Git

Introducción

Las ramas en Git son una forma de trabajar en diferentes características o correcciones de manera aislada. Esto permite que varios desarrolladores trabajen simultáneamente sin interferir entre sí.

Paso 1: Configuración Inicial

Si no tienes Git instalado, descárgalo desde git-scm.com.

Paso 2: Clonar un Repositorio

Si ya tienes un repositorio, puedes saltarte este paso. Si no, clona uno desde GitHub o cualquier otra fuente:

git clone https://github.com/usuario/nombre-del-repo.git
cd nombre-del-repo

Paso 3: Configuración Inicial

Primero, asegúrate de tener Git configurado en tu computadora y de haber clonado un repositorio en el que trabajar. Puedes verificar tu versión de Git escribiendo:

git --version

Si necesitas instalar Git, descárgalo desde git-scm.com.

Paso 4: Verificar Ramas Existentes

Antes de crear una nueva rama, es útil ver las ramas existentes:

git branch
Paso 5: Crear una Nueva Rama

Para crear una nueva rama, utiliza el siguiente comando:

git branch nombre-de-la-rama
Paso 6: Cambiar a la Nueva Rama

Una vez creada la rama, cambia a ella con:

git checkout nombre-de-la-rama

O, si prefieres crear y cambiar de rama en un solo paso:

git checkout -b nombre-de-la-rama

Paso 7: Realizar Cambios y Confirmar

Ahora puedes realizar cambios en tu proyecto. Una vez que hayas hecho tus modificaciones, añade y confirma los cambios:

git add .
git commit -m "Descripción de los cambios realizados"
Paso 8: Volver a la Rama Principal

Para volver a la rama principal (normalmente llamada main o master):

git checkout main

Paso 9: Fusionar la Nueva Rama

Si deseas fusionar los cambios de tu nueva rama en la rama principal, primero asegúrate de estar en la rama principal:

git checkout main

Luego, fusiona tu rama:

git merge nombre-de-la-rama

Tutorial: Cómo utilizar Pull Request en GitHub

Introducción

Un pull request (PR) es una forma en la que los desarrolladores pueden solicitar la revisión y fusión de sus cambios en un proyecto. Es una herramienta clave para la colaboración en proyectos, ya que permite que otros revisen tu código antes de fusionarlo en la rama principal.

Paso 1: Crear un Pull Request en GitHub

Ahora que tu rama está en el repositorio remoto, dirígete a GitHub y abre el repositorio donde subiste tu rama. Verás un botón que sugiere crear un Pull Request. Haz clic en él y sigue los siguientes pasos:

  1. Selecciona tu rama en el menú desplegable.
  2. Asegúrate de que la rama base (normalmente main) esté seleccionada como el destino.
  3. Escribe una descripción clara de los cambios realizados.
  4. Haz clic en Crear Pull Request.

Paso 6: Revisar y Discutir el Pull Request

Una vez que crees el pull request, otros miembros del equipo podrán revisar tu código, dejar comentarios o sugerencias, y discutir los cambios directamente en GitHub. Tú también puedes responder a los comentarios y hacer ajustes según sea necesario.

Paso 7: Fusionar el Pull Request

Cuando el pull request haya sido aprobado y revisado, podrás fusionarlo en la rama principal. Si tienes permisos, verás un botón Merge Pull Request. Haz clic en él para fusionar los cambios.

Si no tienes permisos de escritura en el repositorio, otra persona del equipo tendrá que hacer la fusión por ti.

Paso 8: Eliminar la Rama

Una vez fusionado el pull request, es recomendable eliminar la rama que ya no necesitas para mantener el repositorio limpio. En GitHub, aparecerá un botón para eliminar la rama después de la fusión, o también puedes hacerlo desde la terminal:

git branch -d nombre-de-la-rama git push origin --delete nombre-de-la-rama
